## Decisions Made

- **Status is derived, not stored:** `upcoming` if `start_date > now`, `completed` if `end_date < now`, `active` otherwise (including null dates). No `status` column in DB.
- **Progress counts completed + cancelled toward completion percentage:** `completion_percentage = round((completed + cancelled) / total * 100)`. Mirrors how Plane treats cancelled issues as "done" for sprint velocity.
- **`addIssuesToCycle` uses `updateMany` (not `upsert`):** Issues already in the cycle just get their `cycle_id` reaffirmed — no error, idempotent. Issues in another cycle get moved to this one (an issue can only be in one cycle at a time per the `cycle_id` FK design).
- **Cycle delete nullifies issue `cycle_id`:** Handled by Prisma `onDelete: SetNull` — no service-layer cleanup needed.
