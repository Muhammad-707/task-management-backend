## Decisions Made
- **Default Prisma client output** (`@prisma/client` in node_modules) rather than a custom
  `src/prisma/generated/` path — simplest, most robust for ESM/NodeNext, and avoids pulling
  generated files into `tsc`/eslint. The pre-existing `generated/` gitignore lines are now unused
  but harmless.
- **Temporary `PrismaInitPlaceholder` model** — purely to satisfy Prisma 5's "no models" refusal so
  Phase 0 can generate a client; explicitly throwaway, no migration run against it. (Model names
  can't start with `_`, hence the rename from the first attempt.)
- **`/health` promoted to readiness now** that Prisma is wired (DB ping). Closes the separate
  "GET /health endpoint" roadmap item too.
- **Prisma client connects eagerly at boot** in the plugin — surfaces DB misconfig at startup,
  which is the right failure mode for a server.
